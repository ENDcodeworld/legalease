# 安全政策

## 安全问题报告

我们非常重视 LegalEase 的安全性。如果你发现了任何安全漏洞，请负责任地报告给我们。

### 如何报告

**请勿**在公开的 GitHub Issues 中报告安全漏洞。

请通过以下方式报告：

- **邮箱**: security@legalease.dev（待申请）
- **加密PGP**:
  - Key ID: 0x12345678
  - Fingerprint: ABCD EFGH IJKL MNOP QRST UVWX XYZ12345

### 响应时间

我们会在 **24-48 小时**内确认收到你的报告，并在 **7 天**内提供修复进展。

### 奖励

我们为首次报告有效安全漏洞的研究人员提供：

- 🏆 致谢名单（Hall of Fame）
- 🎁 特别贡献者徽章
- 💰 可选：小额奖金（根据严重程度）

### 范围

本政策适用于：
- 主仓库：github.com/yourusername/legalease
- 所有相关包和依赖
- 演示服务器（如有）

**不包括**：
- 第三方服务（如 OpenAI、Vercel）
- 已公开的漏洞
- DoS 攻击

### 安全最佳实践

对于用户：
1. 妥善保管你的 OpenAI API Key
2. 使用环境变量，不要硬编码密钥
3. 定期更新依赖
4. 配置防火墙规则

对于贡献者：
1. 不要提交任何密钥或敏感信息
2. 遵循最小权限原则
3. 对用户数据保持透明
